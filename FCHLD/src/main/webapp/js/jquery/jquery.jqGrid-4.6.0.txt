원본 소스 변경

1.
함수
htmlDecode, htmlEncode, formatter, addRowData, restoreRow 변경
cellValNonEncode 추가
                        
1) 라인번호 21~30까지 변경
2) 라인번호 1122~1125, 1136~1141, 1144~1149 변경
3) 라인번호 3309~3310 변경
4) 라인번호 9688~9691 변경

변경 사유 : 그리드에서 autoencode 옵션을 true로 주었을 경우 addRowData 나 setRowData 에서
사용자가 임의로 입력한 <input /> <div />등에도 엔코드되어버려 말그대로 깨져서 나오게 된다.
이부분을 엔코드되지 않도록 해주기 위해 코드를 변경했다.
추가로 restoreRow 를 수정한 것은 저장되었던 로우데이터를 다시 화면에 출력시 엔코딩해서 보여주도록 처리한 것이다.

2.
라인번호 2279 변경

변경 내용 : else if( ts.p.sortorder === 'desc') { 를 else 로 변경
변경 사유 : 다중으로 정렬 설정값이 "desc, asc"와 같이 올 때를 위함

3.
라인번호 2303, 2309, 2642 주석처리

변경 사유 : 그리드에서 첫번째 행이 라인번호나 체크박스가 아닐경우 정렬을 하지 않았음에도 첫번째 컬럼에 s-icon을 붙여버리는
문제가 발생하여 이를 처리함. 

4.
라인번호 3197(delRowData), 3370(addRowData) 변경

변경 내용 : jqgfirstrow 클래스가 있는 그리드의 경우 chkFirstRow라는 변수를 이용해 계산식을 살짝 바꾸어 준다.
변경 사유 : 그리드에서 altRows 가 true일 때 로우의 색상 패턴 처리가 되는데 delRowData, addRowData 를 할 때에 총 라인수를
계산해서 패턴처리를 할 때에 높이가 0인 jqgfirstrow 클래스가 들어간 로우는(그리드 헤더와 로우의 사이즈 조절용 tr) 계산에서
제외되었음으로 색상 패턴 처리가 기존가 달라짐으로 이를 다시 포함시켜 계산하도록 변경하였다.
